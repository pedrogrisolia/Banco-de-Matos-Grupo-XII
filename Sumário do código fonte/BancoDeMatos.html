    Quando a Screen3 inicializar, define os campos do Firebase, cria e seleciona o navegador de telas no fundo da tela, exibe a mensagem de boas-vindas, verifica se há acesso a internet, e, caso haja, verifica se há alguma tag (índice) no banco de dados local, caso haja, envia para o banco de dados web do Firebase, e exibe a mensagem para o usuário. No final dessas operações, o banco de dados local é esvaziado.

     

    Evento que é invocado quando o dispositivo perde acesso a internet. Exibe uma mensagem.

    Evento que é invocado quando o dispositivo se conecta a internet. Notifica o usuário e envia os dados (caso haja) do banco de dados local para o banco de dados Web.

    Quando o botão de banco de dados é selecionado no navegador de telas, abre a Screen2.

    Quando o usuário volta para a Screen3, seleciona o item correspondente ao botão de formulário no navegador de telas.

    Quando o sensor de gps obtém um local, se a caixa de texto do local estiver vazia, preenche ela com a localização atual, ou a latitude e longitude, se não houver internet.

    Quando o botão abrirMapa for clicado, esconde todos os elementos visíveis e exibe o mapa do Google Maps. Quando o botão for clicado de novo, faz a operação inversa.

    Quando o mapa estiver renderizado, habilita a localização, compasso, e o MapClickListener que vai aguardar uma ação de clique no mapa.

    Quando o botão localButton for clicado, preenche a caixa de texto do local com o endereço atual do sensor de gps, ou com a latitude e longitude, caso não haja conexão com a internet.

    Quando houver um clique no mapa, pega a latitude e longitude e chama a função GetLocationInfo, que retorna um endereço a partir dessas coordenadas.

    Após o usuário escolher a data, preenche a caixa de texto da data, com a data escolhida com a formatação Day/Month/Year

    Quando a função GetLocationInfo for chamada, pega o seu retorno e preenche na caixa de texto do local.

    Quando o botão enviar for clicado, recolhe todas as informações preenchidas nas caixas de texto e a lista de fotos e armazena no banco de dados web ou no banco de dados local, caso não haja internet. Feito isso, limpa todas as caixas de texto e esvazia a lista de fotos.

    Quando clical no cameraButton, chama a camera do dispositivo para tirar uma foto.

    Após a foto ser tirada, armazena na lista midiaLocal, caso não haja conexão com internet, ou faz upload na nuvem do Cloudinary, caso haja internet.

    Após o usuário escolher uma mídia do dispositivo, armazena na lista midiaLocal, caso não haja conexão com internet, ou faz upload na nuvem do Cloudinary, caso haja internet.

    Quando o método UploadMedia da API do Cloudinary foi chamada, informa se o upload foi bem sucedido ou não, e, caso seja, adiciona a URL da mídia na lista midiaCloudinary.
